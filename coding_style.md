# 代码风格基本要求

* 一定要初始化所有的类成员
* 注释一般用//，写在被注释代码的上面，不要和代码写在同一行；大块的注释可以用/* */
* 头文件包含：<>用于包含操作系统和工具链提供的头文件，""用于包含项目或者第三方库提供的头文件
* 头文件包含的顺序：项目文件，第三方文件，系统文件
* 文件命名：如果是类定义头文件，用大写开头的驼峰形式；如果不是类定义文件，用小写加下划线命名
* 缩进用4个空格
* 左大括号与上一个语句在同一行，不自己单独起一行
* 类名首字母大写，变量名和函数名都用驼峰形式，首字母小写
* 全局变量加g前缀；静态变量加s前缀
* 全局函数和静态成员函数首字母大写
* 函数参数如果与变量重名，不推荐加_来区分，因为太接近了不容易看出错误，成员变量加前缀my
* 指针和引用符号靠近类型名而不是变量名：int& p; 不是 int &p;
* 最大行宽为100
* 文件需要以换行结尾，这个的原因是：如果文件不以换行结尾，下次如果在后面增加内容，会造成原来的
  最后一行增加一个换行符，从而进入diff, 这是没有必要的。
* namespace不产生缩进
* 类成员顺序：变量在前，函数在后，signals放在最后。访问控制的顺序：public, protected, private
* 关于指针和引用：在能使用引用的地方使用引用，因为可以简化语法
* 不定义全局对象，因为它们的构造在main之前进行，且无法控制顺序
* 在头文件中如果只需要一个类的指针，则不要包含整个头文件，而采用前置声明的方式，这样可以减少由于修改头文件而要重新
  编译的文件个数。
* 不要在模块里使用会被别的模块引用的静态全局变量。因为动态库里的全局变量在Windows和Linux下的行为不一样，容易出错。
* Qt signal的参数类型尽量不要用引用，即使注册了类型，Qt也不认为引用类型是注册过的。
